# Website Concept

This project is a modern handbook-style website about Indonesian vehicle periodic inspection (KIR / Uji Berkala Kendaraan Bermotor).

## Core Direction

The website should feel like:
- modern editorial handbook
- practical public guide
- field-oriented technical documentation
- calm and professional
- typography-focused
- mobile-friendly

The website should NOT feel like:
- developer documentation
- startup landing page
- government portal
- SEO content farm
- flashy magazine
- overly personal blog

## Content Philosophy

The content is written from the perspective of a vehicle inspector and field practitioner.

Content should:
- prioritize practical explanations first
- use regulations as context
- explain real-world implications
- avoid raw legal wording without explanation
- avoid SEO filler and clickbait
- feel useful for ordinary people

Target readers:
- vehicle owners
- transport operators
- general public
- people confused about KIR procedures

## Writing Tone

Tone should be:
- calm
- practical
- grounded
- informative
- human
- professional

Avoid:
- corporate startup tone
- excessive jargon
- overdramatic writing
- AI-generic wording

## Article Structure

Typical structure:
1. Pengantar
2. Dasar Regulasi
3. Penjelasan Praktis
4. Hal yang Perlu Diperhatikan
5. Kesimpulan

## Technical Direction

Current priorities:
1. content accumulation
2. editorial consistency
3. readability
4. mobile UX
5. publishing simplicity

Avoid:
- risky architecture changes
- unnecessary complexity
- overengineered systems
- unstable experiments

## Publishing Workflow

The workflow must remain:
- simple
- stable
- low-friction
- mobile-friendly

GitHub mobile publishing is important.

## Visual Direction

Preferred visual characteristics:
- editorial typography
- restrained UI
- subtle navigation
- reading-first layout
- comfortable long-form reading
- soft visual hierarchy
- quiet professional atmosphere

The content should always dominate the UI.
